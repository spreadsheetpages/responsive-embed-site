Which cells am I allowed to edit without breaking the template?
Non-editable cells that are automated or have built-in formulas are usually grayed out. It is generally advised to avoid changing cells with formulas.
Why am I seeing a #DIV/0! or #VALUE! error in the spreadsheet?
[ #DIV/0! ] may appear when the formula in the cell tries to divide by zero. Other errors such as [ #VALUE! ] may appear when the formula uses the wrong type of input (e.g., text instead of numbers). It is recommended to review your input and make sure required fields are filled in correctly.
How do I protect certain cells or sheets from being edited by mistake?
In Google Sheets: Right-click on the tab → Protect Sheet → Set permissions. In Excel: Select the cells you want to allow editing → Format Cells → Protection → Uncheck “Locked” → Then protect the sheet via Review → Protect Sheet.
How do I add more rows or columns to the spreadsheet?
Typically, the template should provide amples rows/columns for the average use. If you're a more advanced user, or know that you'd like to add more rows and columns, simply right-click on the row number or column letter where you want to insert → Choose Insert row above/below or Insert column left/right. Not that some functionality may be affected, so you may want to make a copy to use as a test environment.
Can I undo a change if I make a mistake?
Yes. Press Ctrl+Z (or Cmd+Z on Mac), or click Edit > Undo. In Google Sheets, you can also try using File > Version history to restore earlier versions.
Is there a way to get notifications if someone else (with access) edits the spreadsheet?
Yes. Go to Tools > Notification settings, then choose to be notified when changes are made or when someone submits a form (if linked to one).
What should I do if I accidentally delete something important?
Use Undo (Ctrl+Z or Cmd+Z) immediately. In Google Sheets, you may also try going to File > Version history > See version history to restore a previous version of the spreadsheet.
